# AI21 Labs Provider

## Overview

AI21 Labs is an AI company known for building the Jamba family of models, which use a novel SSM-Transformer hybrid architecture (Mamba-based). The AI21 provider in HelixAgent uses their Studio API, which follows an OpenAI-compatible chat completions format. Jamba models are distinguished by their extremely large context windows (up to 256K tokens) and efficient long-context processing.

## Authentication

AI21 uses the standard Bearer token authentication.

| Header | Value |
|--------|-------|
| `Authorization` | `Bearer <your_api_key>` |

### Step 1: Create an AI21 Account

1. Visit [studio.ai21.com](https://studio.ai21.com)
2. Sign up for an account
3. Complete the verification process

### Step 2: Generate an API Key

1. Navigate to **API Keys** in your AI21 Studio dashboard
2. Create a new API key
3. Copy the key immediately

### Step 3: Store Your API Key

```bash
export AI21_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxx
```

## Configuration

Add the following to your `.env` file or environment:

```bash
# Required
AI21_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxx

# Optional - Override default settings
AI21_BASE_URL=https://api.ai21.com/studio/v1/chat/completions
AI21_MODEL=jamba-1.5-large
```

### Configuration Options

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `AI21_API_KEY` | Yes | - | Your AI21 Studio API key |
| `AI21_BASE_URL` | No | `https://api.ai21.com/studio/v1/chat/completions` | API endpoint URL |
| `AI21_MODEL` | No | `jamba-1.5-large` | Default model to use |

## Supported Models

Models are discovered dynamically via 3-tier discovery (API at `https://api.ai21.com/studio/v1/models`, models.dev, fallback). Fallback models include:

**Jamba Models (current generation)**
- `jamba-1.5-large` - Jamba 1.5 Large (default)
- `jamba-1.5-mini` - Jamba 1.5 Mini (smaller, faster)
- `jamba-instruct` - Jamba Instruct

**Jurassic Models (legacy)**
- `j2-ultra` - Jurassic-2 Ultra
- `j2-mid` - Jurassic-2 Mid
- `j2-light` - Jurassic-2 Light

## Capabilities

- Chat completion: Yes
- Streaming: Yes
- Tool/function calling: Yes
- Vision: No
- Embeddings: No (not via this provider)
- JSON mode: Yes (`response_format` with `type: "json_object"`)
- Code completion: Yes
- Code analysis: Yes
- Text generation: Yes
- Reasoning: Yes

## API Endpoint

| Endpoint | URL |
|----------|-----|
| Chat completions | `https://api.ai21.com/studio/v1/chat/completions` |
| Models list | `https://api.ai21.com/studio/v1/models` |

## Rate Limits

Rate limits depend on your AI21 Studio subscription tier. Check your AI21 dashboard for current limits.

### Model Context Limits (as configured in HelixAgent)

| Limit | Value |
|-------|-------|
| Max tokens (context window) | 256,000 |
| Max input length | 256,000 |
| Max output length | 4,096 |
| Max concurrent requests | 100 |

## Known Limitations

- **No vision support**: AI21 models do not support image inputs.
- **Legacy Jurassic models**: The J2 models use an older architecture and may not support all features (e.g., tool calling). Prefer Jamba models.
- **Health check uses models endpoint**: The health check queries the models list API rather than performing a completion, so it does not consume tokens.
- **API path includes `/studio/v1/`**: The AI21 API uses `/studio/v1/` in its path, not just `/v1/` like most OpenAI-compatible providers.

## Example Usage

### Go Code Example

```go
package main

import (
    "context"
    "fmt"
    "os"

    "dev.helix.agent/internal/llm/providers/ai21"
    "dev.helix.agent/internal/models"
)

func main() {
    provider := ai21.NewProvider(
        os.Getenv("AI21_API_KEY"),
        "", // Use default base URL
        "", // Use default model (jamba-1.5-large)
    )

    req := &models.LLMRequest{
        ID:     "request-1",
        Prompt: "You are a helpful AI assistant.",
        Messages: []models.Message{
            {Role: "user", Content: "Summarize the key differences between SSM and Transformer architectures."},
        },
        ModelParams: models.ModelParameters{
            MaxTokens:   2048,
            Temperature: 0.7,
        },
    }

    ctx := context.Background()
    resp, err := provider.Complete(ctx, req)
    if err != nil {
        fmt.Printf("Error: %v\n", err)
        return
    }

    fmt.Printf("Response: %s\n", resp.Content)
}
```

### Long-Context Example

Jamba models excel at processing long documents due to their 256K context window:

```go
req := &models.LLMRequest{
    ID:     "request-1",
    Prompt: "You are a document analysis assistant.",
    Messages: []models.Message{
        {Role: "user", Content: longDocumentText + "\n\nSummarize the key points of this document."},
    },
    ModelParams: models.ModelParameters{
        MaxTokens:   4096,
        Temperature: 0.3,
    },
}
```

### Tool Calling Example

```go
req := &models.LLMRequest{
    ID:     "request-1",
    Prompt: "You are a helpful assistant with access to tools.",
    Messages: []models.Message{
        {Role: "user", Content: "Look up the current stock price of AAPL."},
    },
    Tools: []models.Tool{
        {
            Type: "function",
            Function: models.ToolFunction{
                Name:        "get_stock_price",
                Description: "Get the current stock price for a ticker symbol",
                Parameters: map[string]interface{}{
                    "type": "object",
                    "properties": map[string]interface{}{
                        "ticker": map[string]interface{}{
                            "type":        "string",
                            "description": "Stock ticker symbol",
                        },
                    },
                    "required": []string{"ticker"},
                },
            },
        },
    },
    ModelParams: models.ModelParameters{
        MaxTokens: 1024,
    },
}
```

### Streaming Example

```go
streamChan, err := provider.CompleteStream(ctx, req)
if err != nil {
    fmt.Printf("Error: %v\n", err)
    return
}

for chunk := range streamChan {
    fmt.Print(chunk.Content)
}
```

### Custom Retry Configuration

```go
retryConfig := ai21.RetryConfig{
    MaxRetries:   5,
    InitialDelay: 2 * time.Second,
    MaxDelay:     60 * time.Second,
    Multiplier:   2.0,
}

provider := ai21.NewProviderWithRetry(
    apiKey, "", "", retryConfig,
)
```
