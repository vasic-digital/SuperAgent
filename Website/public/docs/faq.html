General Questions

What is HelixAgent?

HelixAgent is an AI-powered multi-provider LLM orchestration platform written in Go. It combines responses from multiple language models using intelligent aggregation strategies, providing:

  • OpenAI-compatible APIs for easy integration
  • Support for 7+ LLM providers (Claude, DeepSeek, Gemini, Qwen, Zai, Ollama, OpenRouter)
  • AI debate capabilities for enhanced reasoning
  • Enterprise-grade monitoring with Prometheus and Grafana
  • 95%+ test coverage for production reliability
Which AI providers does HelixAgent support?

HelixAgent natively supports the following LLM providers:

  • Claude (Anthropic) - claude-opus-4, claude-sonnet-4, claude-3.5-haiku
  • Gemini (Google) - gemini-2.5-pro, gemini-2.5-flash, gemini-2.0
  • DeepSeek - deepseek-chat, deepseek-coder
  • Qwen (Alibaba) - qwen-max, qwen-plus, qwen-turbo
  • Zai - zai-pro, zai-lite
  • Ollama - Local LLM deployment (llama2, mistral, codellama)
  • OpenRouter - Unified access to 20+ additional models
Is HelixAgent compatible with OpenAI's API?

Yes! HelixAgent provides an OpenAI-compatible API, which means you can use existing OpenAI SDKs and client libraries with minimal changes. Simply point your OpenAI client to HelixAgent's endpoint and use the helixagent-ensemble model for intelligent multi-provider aggregation.

What is the AI debate system?

The AI debate system allows multiple AI models to discuss and reason together on complex problems. Key features include:

  • Configure 2-10 AI participants with distinct roles and personalities
  • LLM fallback chains for reliability (primary + backup models)
  • Multiple debate strategies: round-robin, structured, adversarial, collaborative
  • Consensus mechanisms with configurable thresholds
  • Cognee AI integration for enhanced response quality
  • Comprehensive metrics and quality scoring

Installation & Setup

What are the system requirements?

HelixAgent requires:

  • Go 1.23+ for building from source
  • PostgreSQL 15 for database storage
  • Redis 7 for caching
  • Docker (optional) for containerized deployment
  • At least one configured LLM provider API key
How do I install HelixAgent?

You can install HelixAgent using Go or Docker:

Using Go:

git clone https://github.com/helixagent/helixagent.git
cd helixagent
make build
./helixagent --config configs/development.yaml

Using Docker:

docker-compose up -d
How do I configure API keys for different providers?

Set API keys as environment variables:

export CLAUDE_API_KEY=your_claude_key
export DEEPSEEK_API_KEY=your_deepseek_key
export GEMINI_API_KEY=your_gemini_key
export QWEN_API_KEY=your_qwen_key
export OPENROUTER_API_KEY=your_openrouter_key

Or configure them in your YAML configuration file using ${ENV_VAR} syntax.

Usage & Features

What is ensemble mode and how does it work?

Ensemble mode (helixagent-ensemble) sends your request to multiple LLM providers simultaneously and aggregates their responses using intelligent strategies:

  • Confidence-weighted voting - Responses weighted by confidence scores
  • Majority vote - Most common response wins
  • Quality-weighted - Best quality response selected
  • Consensus - Requires high agreement among providers

This improves accuracy and reliability by leveraging the strengths of multiple models.

Does HelixAgent support streaming responses?

Yes! HelixAgent supports streaming responses for all supported providers. Set "stream": true in your request to enable streaming. The response will be returned as Server-Sent Events (SSE) in OpenAI-compatible format.

Can I use local LLMs with HelixAgent?

Yes! HelixAgent supports Ollama for local LLM deployment. Configure Ollama by setting:

export OLLAMA_ENABLED=true
export OLLAMA_BASE_URL=http://localhost:11434
export OLLAMA_MODEL=llama2

You can run models like llama2, mistral, codellama, and more locally without sending data to external APIs.

What are the LLM optimization features?

HelixAgent includes 8 optimization tools:

  • GPTCache - Semantic caching with vector similarity
  • Outlines - Structured JSON output generation
  • Enhanced Streaming - Word/sentence buffering, progress tracking
  • SGLang - RadixAttention prefix caching (GPU required)
  • LlamaIndex - Document retrieval with HyDE and reranking
  • LangChain - Task decomposition and ReAct agents
  • Guidance - CFG-based constrained generation
  • LMQL - Query language with declarative constraints

Troubleshooting

I'm getting authentication errors. What should I check?

If you're getting 401 errors:

  • Verify your JWT token is valid using GET /v1/auth/me
  • Check that the token hasn't expired
  • Ensure you're including the Authorization: Bearer TOKEN header
  • Try refreshing your token with POST /v1/auth/refresh
A provider is not responding. How do I debug this?

To debug provider issues:

  • Check provider health: GET /v1/providers/[name]/health
  • View all provider status: GET /v1/health
  • Verify API keys are correctly set
  • Check network connectivity to the provider
  • Review rate limiting settings
  • Enable debug logging: --log-level debug
I'm hitting rate limits. What can I do?

Rate limits are applied per-user:

  • Check your current limits in response headers (X-RateLimit-*)
  • Implement exponential backoff in your client
  • Use semantic caching to reduce redundant requests
  • Consider upgrading to a higher tier for increased limits
  • Batch requests when possible

Is my data secure with HelixAgent?

HelixAgent implements enterprise-grade security measures:

  • JWT-based authentication
  • API key management with encryption
  • Request validation and sanitization
  • Rate limiting to prevent abuse
  • Comprehensive audit logging
  • Security sandboxing for tool execution
  • Environment variable-based secret management
Can I self-host HelixAgent?

Yes! HelixAgent is open-source and designed for self-hosting. You can deploy it:

  • On-premises using Docker Compose
  • On Kubernetes using provided manifests
  • On cloud platforms (AWS, GCP, Azure)
  • With local LLMs using Ollama for complete data privacy
