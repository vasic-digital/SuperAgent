Tutorial 1: Hello World

Get started with your first HelixAgent API call in under 5 minutes.

Prerequisites

  • HelixAgent running locally or deployed
  • At least one LLM provider API key configured
  • curl or any HTTP client

Step 1: Verify Installation

curl http://localhost:8080/v1/health

Expected response:

{
  "status": "healthy",
  "providers": {
    "total": 10,
    "healthy": 8,
    "unhealthy": 2
  }
}

Step 2: Make Your First Chat Request

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "helix-ensemble",
    "messages": [
      {"role": "user", "content": "Hello! What can you do?"}
    ]
  }'

Step 3: Explore the Response

HelixAgent returns an OpenAI-compatible response with additional ensemble metadata:

{
  "id": "chatcmpl-...",
  "object": "chat.completion",
  "model": "helix-ensemble",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "Hello! I'm HelixAgent..."
    }
  }],
  "ensemble": {
    "voting_method": "confidence_weighted",
    "responses_count": 5
  }
}

Tutorial 2: Multi-Provider Configuration

Learn how to configure multiple LLM providers for optimal performance.

Step 1: Set Environment Variables

# Primary providers (API Key)
export DEEPSEEK_API_KEY=your-key
export GEMINI_API_KEY=your-key
export OPENROUTER_API_KEY=your-key

# OAuth providers (from CLI tools)
export CLAUDE_CODE_USE_OAUTH_CREDENTIALS=true
export QWEN_CODE_USE_OAUTH_CREDENTIALS=true

Step 2: Start HelixAgent

./helixagent serve

Step 3: Verify Provider Discovery

curl http://localhost:8080/v1/providers

Tutorial 3: Running an AI Debate

Experience the power of AI Debate with multiple LLMs discussing a topic.

Step 1: Create a Debate

curl -X POST http://localhost:8080/v1/debates \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "Should AI systems have consciousness?",
    "rounds": 3,
    "enable_multi_pass_validation": true
  }'

Step 2: Monitor Progress

curl http://localhost:8080/v1/debates/{debate_id}/status

Step 3: View Results

curl http://localhost:8080/v1/debates/{debate_id}/results

Tutorial 4: Streaming Responses

Enable real-time streaming for responsive AI applications.

Enable Streaming

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Accept: text/event-stream" \
  -d '{
    "model": "helix-ensemble",
    "stream": true,
    "messages": [
      {"role": "user", "content": "Write a poem about AI"}
    ]
  }'

Tutorial 5: Tool Calling

Use function calling with HelixAgent for AI coding assistants.

Define Tools

curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "helix-ensemble",
    "tools": [
      {
        "type": "function",
        "function": {
          "name": "read_file",
          "description": "Read contents of a file",
          "parameters": {
            "type": "object",
            "properties": {
              "file_path": {"type": "string"}
            },
            "required": ["file_path"]
          }
        }
      }
    ],
    "messages": [
      {"role": "user", "content": "Read the README.md file"}
    ]
  }'
