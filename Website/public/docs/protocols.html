MCP / LSP / ACP

Protocol Integration

Extend HelixAgent with Model Context Protocol (MCP), Language Server Protocol (LSP), and Agent Communication Protocol (ACP).

Supported Protocols

HelixAgent supports three major protocols for enhanced AI capabilities

Model Context Protocol (MCP)

Connect LLMs to external tools, data sources, and services. MCP enables AI models to access real-time data, execute code, and interact with external systems.

  • Tool execution and function calling
  • Resource access (files, databases)
  • Prompt templates and management
  • Server discovery and health monitoring

Language Server Protocol (LSP)

Integrate code intelligence into AI workflows. LSP provides language-aware features for code analysis and development assistance.

  • Code completion and suggestions
  • Diagnostics and error detection
  • Symbol search and navigation
  • Refactoring assistance

Agent Communication Protocol (ACP)

Enable multi-agent collaboration and communication. ACP facilitates agent-to-agent messaging and task coordination.

  • Agent discovery and registration
  • Inter-agent messaging
  • Task delegation and coordination
  • Shared context management

MCP Configuration

Configure Model Context Protocol servers and tools

Configuration File

Add MCP configuration to your config.yaml:

mcp:
  enabled: true
  servers:
    - name: filesystem
      type: stdio
      command: npx
      args: ["-y", "@modelcontextprotocol/server-filesystem", "/data"]

    - name: postgres
      type: stdio
      command: npx
      args: ["-y", "@modelcontextprotocol/server-postgres"]
      env:
        POSTGRES_CONNECTION_STRING: "${DATABASE_URL}"

    - name: github
      type: http
      url: http://localhost:8090
      headers:
        Authorization: "Bearer ${GITHUB_TOKEN}"

API Endpoints

Endpoint Method Description
/v1/mcp/capabilities GET Get MCP server capabilities
/v1/mcp/tools GET List available tools
/v1/mcp/tools/call POST Execute a tool
/v1/mcp/prompts GET List prompt templates
/v1/mcp/resources GET List available resources

Tool Execution Example

curl -X POST http://localhost:8080/v1/mcp/tools/call \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "tool": "read_file",
    "arguments": {
      "path": "/data/example.txt"
    }
  }'

LSP Configuration

Configure Language Server Protocol for code intelligence

Configuration File

Add LSP configuration to your config.yaml:

lsp:
  enabled: true
  servers:
    - name: gopls
      language: go
      command: gopls
      args: ["serve"]

    - name: typescript
      language: typescript
      command: typescript-language-server
      args: ["--stdio"]

    - name: python
      language: python
      command: pylsp

  cache_ttl: 5m
  refresh_interval: 1h

API Endpoints

Endpoint Method Description
/v1/lsp/servers GET List LSP servers
/v1/lsp/execute POST Execute LSP request
/v1/lsp/sync POST Sync LSP servers
/v1/lsp/stats GET Get LSP statistics

Unified Protocol Manager

Manage all protocols through a single interface

API Endpoints

Endpoint Method Description
/v1/protocols/execute POST Execute any protocol request
/v1/protocols/servers GET List all protocol servers
/v1/protocols/metrics GET Get protocol metrics
/v1/protocols/refresh POST Refresh protocol servers
/v1/protocols/configure POST Update protocol configuration

Unified Request Example

# Execute a tool through the unified protocol manager
curl -X POST http://localhost:8080/v1/protocols/execute \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "protocol": "mcp",
    "method": "tools/call",
    "params": {
      "name": "read_file",
      "arguments": {
        "path": "/data/config.json"
      }
    }
  }'
