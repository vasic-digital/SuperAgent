v1.4.0 March 25, 2026

Major platform expansion: 41 extracted modules, 43 LLM providers, 6 new modules (DocProcessor, HelixQA, LLMOrchestrator, VisionEngine, HelixMemory, HelixSpecifier), comprehensive safety and security hardening across SP1-SP5 milestones.

Critical Fixes
  • Fixed goroutine lifecycle safety issues across all handlers with WaitGroup patterns
  • Eliminated race conditions detected by go test -race across the full codebase
  • Fixed deadlocks in provider health monitor, debate orchestrator, and cache systems
  • Resolved memory leaks in streaming responses and background task workers
  • Fixed O(n) cache eviction with O(1) LRU replacement in query optimizer
  • Removed 981+ lines of dead code across 19+ files
  • Constitution synchronized to v1.2.0 with 26 mandatory rules
Test Coverage
  • Expanded from 35 to 41 extracted modules, each with independent test suites
  • 60+ challenge scripts validating real-life use cases with zero false positives
  • 835+ tests in HelixSpecifier across 27 packages (unit, integration, E2E, security, stress, benchmark)
  • 80+ tests in HelixMemory covering fusion pipeline, circuit breakers, and infra bridge
  • Comprehensive stress and chaos engineering tests (10x load, cascade failure, memory pressure)
  • BigData, formatters, memory, observability packages elevated to 80%+ coverage
  • gRPC integration tests (21 tests), debate full-flow tests (7 lifecycle tests)
  • 22 Go-native userflow challenges with dependency graph execution
Safety & Security
  • 7 integrated security scanners: Snyk, SonarQube, Gosec, Semgrep, Trivy, staticcheck, go vet
  • Containerized Snyk scanning (38 tests) and SonarQube scanning (45 tests)
  • Gosec scan: 568 findings analyzed, 22 actionable fixes applied
  • File permissions hardened to 0o600/0o750 across 19 files
  • PII detection and redaction via Security module guardrails engine
  • Red team framework for adversarial testing of AI responses
  • SBOM generation (CycloneDX/SPDX) for supply chain security
Performance
  • HTTP/3 (QUIC) as primary transport with Brotli compression across all HTTP communication
  • Lazy loading and sync.Once initialization for all optional services
  • Semaphore-based concurrency limits for parallel health checks and component initialization
  • Debate performance optimizer: parallel LLM execution, response caching with TTL, early termination on consensus
  • Resource limits enforcement: GOMAXPROCS=2, nice -n 19, ionice -c 3 for all tests
  • pprof memory profiling integration for leak detection
  • Benchmark baseline capture for performance regression detection
Documentation
  • Complete documentation for all 41 modules: CLAUDE.md, AGENTS.md, README.md, docs/
  • 61+ video courses covering all major features and subsystems
  • 20+ user manuals with step-by-step guides
  • Architecture diagrams (PlantUML, Mermaid) for all major subsystems
  • Website updated with current feature set, provider list, and changelog
  • Constitution auto-update system with background filesystem watcher
